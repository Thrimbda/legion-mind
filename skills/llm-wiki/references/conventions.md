# 约定

本文中的 `index.md` / `log.md` 都表示默认 baseline，或宿主 schema 明确声明的等价导航 / 日志机制。

## 导航机制（`index.md` 或宿主等价机制）

- 定位：wiki 内容目录，不是时间线。
- 用法：先读它，再决定进入哪些页面。
- 若宿主 schema 已定义等价导航页或索引机制，以宿主约定为准。
- 只有当宿主显式声明其职责、目标位于可写 scope、且允许字段与写法已定义时，才允许写回；否则只能读取，不得回写。
- 最小格式：
  - 页面链接；
  - 一句话说明；
  - 可选分类或元数据。
- 维护要求：
  - 当结构或可导航内容发生变化时同步更新；
  - query 若新增沉淀页、重命名、合并或分类变化，也要同步更新；
  - 页面删除、合并、重命名后保持目录可导航。

## 日志机制（`log.md` 或宿主等价机制）

- 定位：追加式时间线。
- 原则：只追加，不覆盖历史。
- 只有当宿主显式声明其职责、目标位于可写 scope、且允许字段与写法已定义时，才允许写回；否则只能读取，不得回写。
- 安全基线：默认只记录 source/page ID、动作类型、安全摘要与授权依据；不默认复制敏感标题、原文摘录、密钥、令牌、个人数据、内部路径或访问路径。
- 推荐标题前缀：
  - `## [YYYY-MM-DD] ingest | <source-id-or-safe-title>`
  - `## [YYYY-MM-DD] query | <page-id-or-safe-title>`
  - `## [YYYY-MM-DD] lint | <scope-or-safe-id>`
- 每条记录至少包含：
  - 本次动作类型；
  - 触发来源 / 授权依据；
  - 涉及的页面 ID 或 source ID；
  - 关键变更摘要；
  - 后续待办或待验证项（如有）。
- 建议把“证据引用”“变更动作”“待验证推测”分栏书写，避免把推测混成审计事实。
- 发现误记、敏感信息泄露或错误归因时，通过追加更正/撤销记录修正，不覆盖旧记录。

## citation 约定

- 每个稳定结论都应能回溯到 source summary、原始 source，或宿主认可的等价证据节点。
- 页面正文承载解释与综合；日志只保留安全 ID 与动作摘要。
- 若宿主定义引用格式（脚注、内链、frontmatter 字段、Dataview），以宿主为准。
- 未能追溯到来源的内容，只能标为 open question / needs-verification，不写成确定性事实。

## 状态标记

- 推荐使用宿主已有状态体系；宿主缺失时，可采用以下最小语义：
  - `confirmed`：已有足够来源支持
  - `needs-verification`：线索存在但证据不足
  - `contested`：多个来源相互冲突
  - `superseded`：旧结论已被更新证据取代
- 状态标记服务于 lint 与维护，不是装饰性标签。

## 命名与页面落点

- 页面命名遵循宿主 schema；本 skill 不强制具体目录树。
- 优先复用已有 page family；新增页面前先确认是否已有等价页面承载。
- 标题、slug、ID、别名若存在约定，需保持一致，不混用多套命名规则。

## 页面与互链

- 新信息优先并入已有页面；只有在已有页面无法承载时才新增页面。
- 页面新增后尽快补至少一条合理互链，避免成为 orphan page。
- 发现多页重复承载同一概念时，优先整合并保留清晰跳转。
- 页面至少应向上连到导航入口或汇总页，向侧面连到相关 entity / topic / source summary。

## 证据与断言

- 区分“来源事实”“wiki 综合判断”“待验证推测”。
- 新 source 与旧结论冲突时：
  - 不静默覆盖；
  - 标记冲突来源；
  - 记录当前较强证据与仍待确认部分。
- 若 query 被授权写回，页面内容也必须保持可追溯，不能只保留聊天语气结论。
- log 里引用事实时优先写 source/page ID，再让正文页面承载详细内容；日志不是原文摘录仓库。

## 冲突处理

- 先保留可追溯的冲突双方，再判断是否需要 contested / superseded 标记。
- 证据不足时不裁定胜负，只创建 maintenance 项或降级断言强度。
- 若宿主要求单独的冲突页、勘误页或 maintenance backlog，以宿主流程为准。

## 日志安全基线

- 默认不写入敏感摘录、凭证、隐私数据、内部路径、附件真实地址。
- 对附件 / 图片，只记录安全 ID、用途与处理状态；具体路径只在宿主明确允许时引用。
- 例：优先使用 `src-2026-04-article-foo`、`page-comparison-foo-vs-bar` 这类安全 ID；不要直接写本地绝对路径、内部 URI、共享盘地址。
- 误写敏感信息时，通过追加更正记录和后续清理流程处理，不在日志中再次扩散内容。

## 最小自检

在交付一次 ingest / query 写回 / lint 后，至少确认：

1. raw sources 未被改写；
2. 相关 wiki 页面已更新或明确说明为何未更新；
3. `index.md` 或宿主等价导航机制已反映新增页面或结构变化；
4. `log.md` 或宿主等价日志机制已追加新记录；
5. 新增页面不存在明显孤立、无来源支撑或无互链问题；
6. 新日志未写入敏感原文，且授权依据/更正方式清晰。
